# 企业数据 DataAgent 智能体

## 赛题描述

本赛题要求参赛者基于 [Spring AI Alibaba](https://github.com/alibaba/spring-ai-alibaba) 框架，社区已经构建了 DeepResearch、NL2SQL、RAG 等垂直方向的原子能力，进一步优化 DataAgent 的整体运行效果，提升报告的全面性、准确性与参考价值。同时，需重点增强在典型业务场景（如销量统计、根因分析等）中的报告生成能力。技术架构层面，作品需支持 DeepResearch、RAG 等多数据源接入，推动 DataAgent 在真实业务场景中的实用性与智能化水平。


### 技术要求与架构参考：


深度研究（DeepResearch）： 利用 Spring AI Alibaba Deep Research 能够进行深入、多源的信息收集和综合，通常涉及非结构化或外部数据，例如网络信息、学术论文和新闻报道。它超越了简单的关键词搜索，实现了迭代的、基于推理的探索。

NL2SQL： 利用 Spring AI Alibaba Deep NL2SQL 允许用户使用日常语言与结构化内部数据库（如关系型数据库、数据仓库）进行交互，从而抽象化了SQL语法和数据库模式的复杂性。

RAG：利用 Spring AI Alibaba 提供的 RAG 知识库向量化、检索等能力，可以很好的整合企业内的非结构化、多模态数据。

DataAgent 重点探索方向：在本赛题中，选手需实现一个具备自然语言到数据智能分析能力的 DataAgent。该智能体应能够基于 NL2SQL 技术，将用户自然语言问题自动转化为结构化数据库查询，实现对海量数据的快速检索与计算。同时，需结合 DeepResearch 能力，支持跨多源数据进行深度推理与综合分析，确保结果的准确性与洞察力。为增强知识覆盖面，要求引入 RAG（检索增强生成） 方案，在回答过程中动态调用外部文档与业务数据，提升数据理解与解释能力。此外，DataAgent 需具备 **自主规划** 能力，能够根据复杂任务的语义需求自动进行分解、选择合适工具并组织执行路径。整体目标是构建一个高效、智能、可扩展的数据分析代理，推动自然语言驱动的数据洞察与业务决策。
